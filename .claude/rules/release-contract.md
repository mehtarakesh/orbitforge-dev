# Release Contract Rule

OrbitForge is supposed to help people ship, not just generate attractive text.

Protect the response contract:

1. concise plan
2. implementation approach
3. validation steps
4. remaining risks

When editing prompts, UI copy, or output handling:

- keep validation visible
- keep risk visible
- avoid replacing actionable structure with generic assistant phrasing
- prefer outputs that can be acted on by a working developer immediately

If a change makes the tool sound smarter but makes verification weaker, it is probably a regression.
